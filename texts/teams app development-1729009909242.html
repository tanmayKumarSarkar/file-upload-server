App Localization in Microsoft Teams

  1. Localize Your AppSource Listing:

    • Provide metadata (descriptions, screenshots, name) in the languages you want your app to be listed in. Specify these languages on the Marketplace listings page in Partner Center1.

  2. Localize Strings in Your App Manifest:

    • Use the Microsoft Teams app schema to add the localizationInfo property in your manifest.json file. This includes the default language and additional languages your app supports1.

  3. Handle Localized Text Submissions:

    • Ensure your app can handle text submissions in different languages, which may involve setting up appropriate encoding and storage mechanisms1.

Adaptive Card Actions in Microsoft Teams

  1. Universal Actions:

    • Use Action.Execute to handle actions across different platforms like Teams and Outlook. This ensures a consistent user experience and simplifies action handling2.

  2. User-Specific Views:

    • Customize the view and actions on Adaptive Cards based on the user interacting with them. For example, different users can see different buttons or information on the same card2.

  3. Sequential Workflow Support:

    • Implement workflows where actions on cards trigger subsequent steps, maintaining the context and state across interactions2.

App Authentication in Microsoft Teams

  1. Single Sign-On (SSO):

    • Enable SSO to use the user’s Teams identity for accessing your app. This simplifies the login process and enhances security3.

  2. Nested App Authentication (NAA):

    • Use NAA for SPAs embedded in Teams, Outlook, and Microsoft 365. This model supports incremental and dynamic consent for permissions, improving security and user experience4.

  3. Third-Party OAuth Providers:

    • Integrate with third-party OAuth providers like Google, Facebook, or GitHub to authenticate users. This can be useful if your app needs to support multiple identity providers3.
